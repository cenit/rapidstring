◆ rs_free()

RS_API void rs_free ( rapidstring *  s)

Frees a string.

The string is in an invalid state after freeing. You must call rs_init() if you wish to reuse the same string.

A jump may be avoided by directly calling RS_FREE(s->heap.buffer) if the string is known to be on the heap.

Calling this fuction is unnecessary if the string size is always smaller or equal to RS_STACK_CAPACITY.

Parameters
[in]sThe string to free.
Allocation
Never.
Since
1.0.0
